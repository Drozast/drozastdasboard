Particiones:

• Asignado: 437GB (expandido al máximo)
• Espacio utilizado dinámicamente por Docker y servicios

Cache SSD (SDB)

SSD 120GB - bcache

Activo
Estado
111.8GB
Capacidad
Writeback
Modo
Dispositivo: /dev/sdb (111.8GB)

Función: Cache SSD para el RAID1
• Acelera operaciones de lectura/escritura
• Usa tecnología bcache del kernel Linux
• Dispositivo resultante: /dev/bcache0

Beneficio: El RAID HDD obtiene velocidades cercanas a SSD para datos frecuentes

RAID 1 Datos (SDC + SDD)

2x 4TB HDD en Espejo

--

--

--%
Uso
Dispositivos:
• /dev/sdc - 3.6TB (Disco 1)
• /dev/sdd - 3.6TB (Disco 2)

RAID: /dev/md0 (linux_raid_member)
• Tipo: RAID 1 (Espejo/Mirror)
• Los datos se duplican en ambos discos
• Tolerancia: Puede fallar 1 disco sin pérdida

Cache: Usa /dev/sdb via bcache
Montaje: /mnt/nextcloud-data
Uso: Nextcloud, backups, multimedia

# ─────────────────────────────────────────────────────────────────────────────────
# MAPAS SELF-HOSTED (Alternativa a Google Maps - Solo Chile)
# ─────────────────────────────────────────────────────────────────────────────────
# Stack: TileServer GL + Nominatim + OSRM
# Datos: OpenStreetMap Chile (~300MB PBF, ~500MB mbtiles)
# Formato: Vector tiles (OpenMapTiles schema, protobuf)
# RAM estimada: ~4-5 GB total para los 3 servicios
# Docker compose: ~/maps/docker-compose.yml
# Puertos: TileServer=8082, Nominatim=8081, OSRM=5000

# CAPAS DISPONIBLES EN VECTOR TILES (OpenMapTiles):
# place            → Ciudades, pueblos, aldeas (z0-14)
# transportation   → Calles, carreteras, vias (z4-14)
# transportation_name → Nombres de calles (z8-14)
# poi              → Puntos de interes: comercios, hospitales, etc (z12-14)
# housenumber      → Numeros de casa (z14)
# building         → Edificaciones (z13-14)
# water            → Rios, lagos, oceano (z6-14)
# water_name       → Nombres de cuerpos de agua (z14)
# waterway         → Cursos de agua (z8-14)
# boundary         → Limites administrativos (z0-14)
# landuse          → Uso de suelo: residencial, comercial (z4-14)
# landcover        → Cobertura: bosque, pasto, nieve (z0-14)
# park             → Parques y areas verdes (z11-14)
# aeroway          → Pistas de aeropuerto (z11-14)
# aerodrome_label  → Nombres de aeropuertos (z10-14)
# mountain_peak    → Cumbres y montanas (z11-14)

# ENDPOINTS DISPONIBLES:
